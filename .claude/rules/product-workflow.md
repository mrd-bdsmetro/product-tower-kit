# Product Tower Workflow Rules

## Core Workflow

1. **Initialize** → `/pt:init` or `npx product-tower init`
2. **Research** → T0 market research (Express or Pro mode)
3. **Foundation** → T1-T3 segmentation
4. **Discovery** → T4-T6 user personas + needs
5. **Anti-Bias** → AB1-AB4 mandatory before T7
6. **Validate** → T7 PMF gate (≥30/50)
7. **Scope** → T8-T9 features + user stories
8. **Offer** → T9.5 pricing + guarantee
9. **Handoff** → T10-T14 (external skills)

## Gate Rules

- Each tier requires previous tier completion
- T7 requires AB1-AB4 (or force skip with penalty)
- PMF(adjusted) < 30/50 = BLOCKED at T8
- Force skip = risk acknowledged, output marked ⚠️

## Anti-Bias Rules

- AB1: Counter-search with NEW keywords (not from T0)
- AB2: Red team PMF signals
- AB3: Field observation (digital or physical)
- AB4: User interview (min 5 people, Mom Test)
- AB5: Strategic analysis (optional, +0.5 bonus)
- AB6: Founder insight (mandatory for solo founder, +1.0 bonus)

## Confidence Rules

- Every data point must have confidence tag
- 🤖 AI inference: 60%
- 📊 Cited research: 80%
- 👤 User interview: 90%
- ✅ Validated by usage: 95%
