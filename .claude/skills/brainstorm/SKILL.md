# Brainstorm — Product Decisions

## Goal
Brainstorm product solutions with trade-off analysis. Brutal honesty. No fluff.

## Core Principles
- **YAGNI** — You Ain't Gonna Need It
- **KISS** — Keep It Simple, Stupid
- **DRY** — Don't Repeat Yourself
- **Ship > Perfect** — 1 session = 1 output

---

## Process

### 1. Discovery
- Ask clarifying questions
- Understand constraints
- Identify success criteria

### 2. Scope Assessment
- Is this BUILD / SELL / SYSTEM?
- Who pays? (TÊN cụ thể)
- Evidence of demand? (DATA)
- 1 person + ADHD, ship 2 weeks?

### 3. Research
- Existing solutions?
- Competitor analysis?
- User feedback?

### 4. Analysis
- Trade-off matrix
- Pros/cons quantified
- Second-order effects

### 5. Debate
- Challenge assumptions
- Surface alternatives
- Anti-rationalization check

### 6. Consensus
- Pick 1-2 options
- Document decision rationale

### 7. Documentation
- Output: decision doc with trade-offs
- Next steps

---

## Anti-Rationalization Table

| Excuse | Reality Check |
|--------|---------------|
| "It's just a small feature" | Scope creep kills. Define MVP. |
| "Users will love this" | Did you ask? Show evidence. |
| "We need this before launch" | Do we? Ship without it first. |
| "It's technically interesting" | So what? Does it drive revenue? |
| "Competitors have it" | So what? Are users asking for it? |
