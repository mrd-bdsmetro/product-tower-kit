# Documentation Workflow - Auto-Docs Generation

## Overview

Documentation should be **generated from code**, not written manually. This workflow defines how docs are created, maintained, and kept in sync with code changes.

**Trigger:** Code change → Auto-generate → Human review → Publish

---

## Doc Types

| Type | Generated by | Updated when |
|------|--------------|--------------|
| API Reference | Code (JSDoc, OpenAPI) | On build |
| README | Templates + manifest | On release |
| Changelog | Git commit messages | On merge |
| Architecture | Doc blocks in code | On change |
| Guides | Human + AI hybrid | On need |

---

## Automated Generation

### API Docs (OpenAPI/Swagger)

```bash
# Generate from code annotations
npm run docs:api

# Output: docs/api/openapi.json, docs/api/index.html
```

**Triggers:**
- New endpoint added
- Request/response shape changed
- Schema updated

### README Generation

```bash
# Generate from package.json + manifest
npm run docs:readme

# Updates: Installation, Usage, Commands, License
```

**Triggers:**
- New command added
- Dependency changed
- Version bump

### Changelog Generation

```bash
# Generate from conventional commits
npm run docs:changelog

# Output: CHANGELOG.md with categorized changes
```

**Triggers:**
- Merge to main
- Release tag created

## Documentation Standards

### Required sections for guides

1. **Overview** - What this does, why it matters
2. **Prerequisites** - What you need before starting
3. **Step-by-step** - Numbered steps, clear commands
4. **Examples** - Code snippets for common cases
5. **Troubleshooting** - Common errors + fixes
6. **Related** - Links to related docs

### Metadata frontmatter

```markdown
---
status: draft|review|published
type: guide|api|reference|architecture
owner: team-member-name
last_updated: YYYY-MM-DD
tags: [relevant, tags]
review_cycle: monthly|quarterly|on-change
---
```

## Review Cycle

| Doc type | Review frequency |
|----------|------------------|
| API reference | On change |
| Guides | Monthly |
| Architecture | Quarterly |
| README | On release |

---

## Versioning

Docs live alongside code in same repo:
```
docs/
  {type}-{name}.md      # Guide: guide-authentication.md
  api/                   # Generated API docs
  architecture/          # System architecture docs
```

Version in frontmatter matches code version.
